# 📱 내 앱 스타터 템플릿 (비개발자용 가이드)

웹앱으로 시작해서, 같은 코드로 **구글 플레이스토어 / 애플 앱스토어**까지 배포할 수 있는 템플릿입니다.
**초기 셋팅은 딱 한 번만** 하면 되고, 그 다음부터는 어느 컴퓨터에서든 / 클로드 모바일에서든
이어서 작업할 수 있어요.

---

## 🧠 핵심 개념 (왜 이렇게 하나요?)

> **GitHub를 "원본 보관함"으로 두는 것**이 전부예요.
> 모든 기기(데스크탑, 다른 컴퓨터, 클로드 모바일)는 GitHub에서 코드를 받아오고,
> 작업한 걸 다시 GitHub에 올립니다. 그래서 "초기 셋팅"을 매번 반복할 필요가 없어요.

```
   데스크탑 ──┐
   다른 컴퓨터 ─┼──▶  GitHub (원본)  ──▶  Vercel (자동으로 웹 배포)
   클로드 모바일 ─┘                    └──▶  앱스토어 (나중에)
```

---

## ✅ 처음 한 번만: 초기 셋팅 (약 15분)

### 1단계 — GitHub에 올리기
1. https://github.com 가입 (무료)
2. 오른쪽 위 `+` → **New repository** → 이름 입력(예: `my-app`) → **Create**
3. 클로드에게 이렇게 말하세요:
   > "이 폴더를 방금 만든 GitHub 저장소에 올려줘"
   - 클로드가 `git` 명령으로 대신 올려줍니다. (직접 안 해도 돼요)

### 2단계 — 템플릿으로 지정 (재사용의 핵심 ⭐)
GitHub 저장소 페이지 → **Settings** → 맨 위 **Template repository** 체크.
- 이제 다음에 새 앱을 만들 땐, 이 저장소에서 **"Use this template"** 버튼만 누르면
  셋팅이 끝난 새 프로젝트가 바로 생깁니다. **셋팅 반복 끝!**

### 3단계 — Vercel에 연결 (웹 자동 배포)
1. https://vercel.com → **GitHub 계정으로 로그인**
2. **Add New → Project** → 방금 만든 저장소 선택 → **Deploy**
3. 끝! 이제부터는 GitHub에 변경사항을 올릴 때마다 **자동으로 웹사이트가 업데이트**됩니다.
   - 주소(예: `my-app.vercel.app`)가 생기고, 누구나 접속할 수 있어요.

## 🔁 매일 작업하는 법 (셋팅 다시 안 함)

### 데스크탑/다른 컴퓨터에서
1. 클로드에게: "내 GitHub의 my-app 저장소 가져와줘(clone)"
2. 클로드에게: "이 프로젝트 셋업하고 실행해줘"
   → 클로드가 `CLAUDE.md`를 읽고 알아서 설치(`npm install`)하고 실행해줍니다.
3. 만들고 싶은 걸 말로 요청 → 다 되면 클로드에게: "GitHub에 올려줘(push)"

### 클로드 모바일에서
1. 같은 GitHub 저장소를 연결
2. "이 화면 이렇게 바꿔줘" 식으로 요청 → 변경 후 push
3. 데스크탑에서 다시 열면 모바일에서 한 작업이 그대로 이어집니다.

> 💡 항상 작업 끝에 **"push(올리기)"** 만 기억하면 됩니다. 그게 모든 기기를 동기화하는 열쇠예요.

## 🏪 앱스토어 배포 (앱이 어느 정도 완성된 뒤)

웹으로 충분히 만든 다음, 같은 코드를 앱으로 포장합니다. (Capacitor가 해줌)

| 필요한 것 | 비용 | 비고 |
|---|---|---|
| 애플 앱스토어 | 개발자 계정 연 $99 | **맥 컴퓨터 필요** |
| 구글 플레이스토어 | 최초 1회 $25 | 윈도우/맥 모두 가능 |

클로드에게 "앱스토어용으로 빌드 준비해줘"라고 하면 `CLAUDE.md`에 적힌 순서대로 도와줍니다.
(처음엔 `npx cap add ios` / `npx cap add android` 한 번, 이후엔 `npm run sync`)

## 📂 이 폴더에 뭐가 들었나요?

| 파일/폴더 | 설명 |
|---|---|
| `app/page.tsx` | **메인 화면**. 여기를 바꾸면 앱 첫 화면이 바뀜 |
| `app/globals.css` | 색깔/디자인 |
| `CLAUDE.md` | 클로드가 자동 셋업하려고 읽는 안내서 (지우지 마세요) |
| `setup.sh` | 한 번에 설치해주는 스크립트 |
| `package.json` | 어떤 라이브러리를 쓰는지 목록 |
| `capacitor.config.ts` | 앱스토어용 앱 이름/ID 설정 |
| `.gitignore` | GitHub에 안 올릴 파일 목록 |
