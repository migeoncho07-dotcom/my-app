# 이 프로젝트를 셋업하는 방법 (클로드가 읽는 안내서)

> 메간님: 이 파일은 **클로드가 자동으로 읽고 셋팅을 대신 해주기 위한** 파일이에요.
> 새 컴퓨터나 클로드 모바일에서 이 프로젝트를 열면, 클로드에게
> "이 프로젝트 셋업해줘"라고만 말하면 아래 내용을 보고 알아서 설치/실행해줍니다.

## 프로젝트 개요
- Next.js(웹) + Capacitor(앱스토어 포장)로 만든 웹앱입니다.
- 웹은 Vercel에 자동 배포되고, 같은 코드를 iOS/안드로이드 앱으로도 만들 수 있습니다.

## 처음 셋업할 때 (클로드가 실행할 명령)
1. 의존성 설치:  `npm install`
   - (또는 준비된 스크립트:  `bash setup.sh`)
2. 개발 서버 실행:  `npm run dev`  → 브라우저에서 http://localhost:3000 확인

## 웹 배포 (Vercel)
- GitHub에 push하면 Vercel이 자동으로 배포합니다. 별도 명령 필요 없음.
- 배포 전 빌드 확인:  `npm run build`

## 앱스토어용 빌드 (나중에)
1. 정적 파일 + 네이티브 동기화:  `npm run sync`
2. iOS 프로젝트 열기(맥 필요):   `npm run open:ios`
3. 안드로이드 프로젝트 열기:      `npm run open:android`
- 최초 1회만:  `npx cap add ios`  /  `npx cap add android`

## 작업 규칙 (클로드에게)
- 화면 코드는 `app/` 폴더에 있습니다. 메인 화면은 `app/page.tsx`.
- 비개발자 사용자입니다. 전문 용어는 풀어서 설명하고, 한 번에 한 단계씩 진행하세요.
- 변경 후에는 항상 git에 commit하고 push하도록 안내하세요 (모든 기기 동기화의 핵심).
